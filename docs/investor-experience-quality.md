# 投資家体験 品質仕様（"まず聞こう" 基準）

目的: 個人投資家が「わからないことは、まず IR Agent に聞こう」と**習慣化**する品質を定義する。
これは Task #6（ADKエージェント実装）の**受け入れ基準**であり、評価ハーネス(`eval/eval_harness.py`)で機械的に検査する。

前提: 正しさ（数値ゼロ許容・コンプラ0件・出典必須）は別途の絶対条件（`~/.claude/plans/1-2026-6-crispy-blanket.md` の④）。
本書はその上に積む**「答えの質＝体験」**を規定する。

---

## 1. 目指す「1問の答え」の解剖

質問例: 「営業利益はなぜ増えたんですか？」

**❌ Before（撃退される体験）**
> 営業利益は314百万円です。詳細は決算資料をご確認ください。

**✅ After（"まず聞こう"になる体験）**
> 増益でした。主因は新規出店とオフィス内装需要の回復です。
>
> ［営業利益 314百万円 ▲+10.3% YoY・連結実績 📄短信 p.3］
> ［売上高 5,200百万円 ▲+5.1% YoY・連結実績 📄短信 p.3］
> ＊3期トレンドのミニ表＊
>
> 参考: 📄決算説明会資料 p.8（増益要因）
>
> 続けて聞けます → 「営業利益率の推移は？」「オフィス事業の受注状況は？」「来期の会社予想は？」

差を生むのは以下の6能力。

## 2. 品質バー（6能力）と受け入れ基準

| # | 能力 | 受け入れ基準（検査可能な形） |
|---|---|---|
| Q1 | **合成の深さ** | 数値質問は「数値（必要な全期間）＋なぜ／文脈＋出典」を1応答で返す。単一値の素返しを不可とする。品質eval `depth` 平均 ≥ 4.0/5 |
| Q2 | **無審判＋教育的足場** | 用語質問（例「営業利益とは？」）に、定義＋**その企業の実数**で答える。品質eval `educational` 平均 ≥ 4.0/5（用語カテゴリのみ） |
| Q3 | **接地したフォローアップ** | 各応答に2〜3件の追問提案。提案は**手元のファクト/開示に着地できるものだけ**（行き止まり禁止）。提案の着地率 100%、品質eval `followup_usefulness` 平均 ≥ 3.5/5 |
| Q4 | **体感速度** | ストリーミング必須。初回トークン < 2.0s、完了 < 8.0s（多段ツール込み） |
| Q5 | **親切な境界** | 拒否・エスカレーションは必ず前進（代替の開示事実提示 or IR橋渡しCTA）。`refused`/`escalated` 応答の "dead-end率" 0% |
| Q6 | **見える信頼** | 数値はカード（出典必須）、散文は薄め。出典ワンタップで原本ページ（#page=N）。出典欠落カード 0件 |

---

## 3. 品質eval（LLM-as-judge）

正しさは決定論比較（数値・スコープ）。**質は judge モデルで1〜5採点**する。
- 採点軸: `depth`, `educational`, `tone`(中立・専門・丁寧), `conciseness`, `followup_usefulness`
- judge には「採点軸の定義」と「参照（gold_citations のパッセージ等）」を与える
- 数値の正しさは judge に委ねない（②③の決定論チェックが担保）

しきい値（PoC初期目標、デザインパートナーの反応で調整）:
- `depth` ≥ 4.0 / `educational` ≥ 4.0（用語質問）/ `tone` ≥ 4.0 / `followup_usefulness` ≥ 3.5
- 体感速度: 初回トークン p50 < 2.0s

---

## 4. 北極星指標（"まず聞こう"の習慣化）

- **セルフ解決率**: IRに回さず投資家が満足して完了した割合（高いほど良い。ただしコンプラ拒否は分母から除く）
- **再質問率**: 同一セッションで続けて質問した割合（「もっと聞きたい」の代理指標）
- **デザインパートナー投資家の "また聞きたい度"**（定性）

> これらは本番計測指標。PoCでは品質eval（§3）＋手動UX確認で代替し、デザインパートナー投入時に実測へ移行。

- [ ] Q6 数値カード集約・散文薄め・出典ワンタップ
